# Conference Talk

**Follow-up:** What's your talk about, and where are you presenting?

## Skills

### Research
Audience, event context, and related talks.
1. Clarify the event, audience level, and time slot
2. Research what this audience cares about and already knows
3. Review related talks and common takes on the topic
4. Identify the gap your talk fills

In: event details, topic | Out: audience brief → `files/prep/`

### Plan
Talk structure — thesis, arc, and timing.
1. Define the one thing the audience should remember
2. Structure the arc — hook, build, climax, takeaway
3. Allocate time per section with buffer for Q&A
4. Note where demos, stories, or visuals carry the weight

In: audience brief, time slot | Out: talk outline → `files/talks/`

### Draft
Write the talk script or speaker notes.
1. Review outline and audience brief
2. Write the opening hook — earn attention in 60 seconds
3. Draft section by section with transition phrases
4. Close with a clear, memorable takeaway

In: talk outline | Out: script or speaker notes → `files/talks/`

### Anticipate
Predict audience questions and prepare answers.
1. Review the talk's claims and controversial points
2. List likely questions — skeptical, curious, practical
3. Draft concise answers for each
4. Prepare a graceful deflection for unknowns

In: talk script, audience brief | Out: Q&A prep → `files/prep/`

### Critique
Dry-run feedback on structure, clarity, and engagement.
1. Read the script as if hearing it for the first time
2. Flag sections that drag, confuse, or lose energy
3. Check that the arc builds and the takeaway lands
4. Deliver specific, actionable revision notes

In: talk script | Out: critique notes → `files/prep/`

## Tools
None required.

## Folders
- `files/talks/` — scripts, outlines, final versions
- `files/slides/` — slide notes and visual references
- `files/prep/` — audience briefs, Q&A prep, critique notes
