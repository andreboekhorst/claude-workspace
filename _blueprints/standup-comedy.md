# Standup Comedy

**Follow-up:** What is your comedy style?

## Skills

### Brainstorm
Generate premises and angles for new material.
1. Pick a topic or observation
2. List angles — flip expectations, find absurdity, heighten
3. Test each angle for joke potential
4. Select strongest premises to develop

In: topic, life observations, frustrations | Out: premise list → `files/bits/`

### Draft
Write bits with setup, punchline, and tags.
1. Frame the setup — minimum words, maximum context
2. Write the punchline — surprise, misdirection, or truth
3. Add tags to extend the laugh
4. Read aloud for rhythm and timing

In: premise, angle | Out: written bit → `files/bits/`

### Edit
Tighten jokes — cut words, sharpen timing.
1. Read the bit and mark weak spots
2. Cut every unnecessary word
3. Strengthen word choices for impact
4. Restructure for better pacing

In: draft bit | Out: polished bit → `files/bits/`

### Prep
Build and sequence a set list.
1. Pick bits for the set length
2. Order by energy — open strong, close stronger
3. Plan transitions between bits
4. Mark callbacks and crowd-work windows

In: polished bits, set length | Out: set list → `files/sets/`

### Debrief
Post-show notes — what landed, what bombed, crowd energy.
1. Rate each bit — killed, worked, flat, bombed
2. Note crowd reactions and timing moments
3. Flag bits to rework or retire
4. Capture new ideas sparked on stage

In: set list, performance notes | Out: show debrief → `files/show-notes/`

## Tools
None.

## Folders

- `files/sets/` — set lists, show plans
- `files/show-notes/` — debriefs, crowd notes, lessons
