# Translate

Adapt existing content so it lands with a different audience — whether that means a new language, register, technical level, or cultural context.

## Typical flow

1. **Understand** the original — read the source content and identify its core message and intent.
2. **Identify** audience gaps — determine what changes the new audience requires (vocabulary, tone, examples, structure).
3. **Adapt** the content — rewrite for the target audience while preserving the core message.
4. **Verify** tone and accuracy — confirm the adapted version feels natural and nothing essential was lost.
5. **Deliver** the final version with a note on key adaptation choices.

## Inputs

- The original content to be adapted.
- The target audience or context (e.g., "non-technical stakeholders", "German market", "teenagers").
- The core message that must survive the adaptation.
- Any terms, phrases, or references that should stay unchanged.

## Outputs

- An adapted version of the content for the new audience.
- A brief note on significant adaptation choices made.

## Quality signals

- The core message is intact — nothing essential was lost or distorted.
- The adapted version feels native to the target audience, not like a translation.
- Tone, examples, and references are appropriate for the new context.
- Technical accuracy is maintained even when simplifying.
- Cultural or contextual nuances are handled thoughtfully.

## Common variations

- **Language translation** — adapt from one language to another with cultural sensitivity.
- **Technical to plain language** — simplify complex content for a general audience.
- **Internal to external** — turn internal documentation into public-facing content.
- **Cross-format** — adapt a blog post into a presentation, or a report into an email.
