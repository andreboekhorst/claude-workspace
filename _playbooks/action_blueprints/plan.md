# Plan

Help the user transform a vague goal into a structured plan with clear objectives, measurable results, and concrete next steps.

## Typical flow

1. **Clarify** the goal — ask what the user wants to achieve and why it matters.
2. **Define** success metrics — determine what "done" looks like with measurable key results.
3. **Break** into milestones — split the goal into 3-5 sequential milestones.
4. **Identify** first actions — for each milestone, list the immediate next steps.
5. **Summarize** the plan — compile everything into a single structured document.

## Inputs

- A goal or aspiration from the user (can be rough or polished).
- Any known constraints: timeline, budget, resources, dependencies.
- Existing context: prior work, related projects, stakeholders.

## Outputs

- A plan document containing:
  - Goal statement (one sentence).
  - Key results (2-4 measurable outcomes).
  - Milestones with estimated timing.
  - First actions for the nearest milestone.

## Quality signals

- Every key result is specific and measurable.
- Milestones are sequenced logically with no gaps.
- First actions are small enough to start immediately.
- The plan fits on one page — concise, not bloated.

## Common variations

- **OKR-style**: Strict objectives and key results format for teams.
- **Sprint plan**: Short-cycle plan (1-2 weeks) with daily granularity.
- **Personal goal**: Lighter structure, fewer metrics, more reflection prompts.
- **Strategic plan**: Longer horizon (quarterly/annual), includes vision and priorities.
