# Action Blueprint Index

Reference outlines for building actions. Each blueprint describes how a type of work typically flows — use them as guidance when creating new actions.

## Research & Analysis
| Blueprint | File | When to use |
|-----------|------|-------------|
| Research | `research.md` | User needs to investigate a topic and deliver organized findings |
| Compare | `compare.md` | User needs to evaluate options side by side |
| Synthesize | `synthesize.md` | User has multiple sources and needs to find patterns across them |
| Verify | `verify.md` | User needs to fact-check claims or validate information |
| Scan | `scan.md` | User wants to survey a domain for trends or emerging signals |

## Writing & Content
| Blueprint | File | When to use |
|-----------|------|-------------|
| Draft | `draft.md` | User needs to write a document from a brief or outline |
| Edit | `edit.md` | User has existing content that needs improvement |
| Translate | `translate.md` | User needs content adapted for a different audience or language |
| Summarize | `summarize.md` | User has long content that needs condensing |
| Plan Content | `plan-content.md` | User needs to map out a content series or calendar |

## Thinking & Decisions
| Blueprint | File | When to use |
|-----------|------|-------------|
| Brainstorm | `brainstorm.md` | User needs to generate and narrow down ideas |
| Decide | `decide.md` | User faces a choice and needs structured thinking |
| Pre-mortem | `pre-mortem.md` | User wants to stress-test a plan before committing |
| Prioritize | `prioritize.md` | User has a list of items and needs to rank them |
| Evaluate | `evaluate.md` | User needs structured pros/cons analysis |

## Planning & Strategy
| Blueprint | File | When to use |
|-----------|------|-------------|
| Plan | `plan.md` | User needs to set goals and define next actions |
| Kickoff | `kickoff.md` | User is starting a new project and needs to scope it |
| Map | `map.md` | User needs to lay out phases and milestones |
| Reflect | `reflect.md` | User wants to review what happened and extract lessons |
| Review | `review.md` | User wants to reflect on a period and plan the next one |

## Operations & Process
| Blueprint | File | When to use |
|-----------|------|-------------|
| Checklist | `checklist.md` | User has a process that should become reusable steps |
| Document | `document.md` | User needs to write a standard operating procedure |
| Prep | `prep.md` | User has an upcoming meeting and needs to prepare |
| Debrief | `debrief.md` | User has raw notes that need structuring into actions |
| Design Form | `design-form.md` | User needs a structured questionnaire or intake form |

## Learning & Communication
| Blueprint | File | When to use |
|-----------|------|-------------|
| Explain | `explain.md` | User needs a complex topic broken down for an audience |
| Teach | `teach.md` | User needs to structure a teaching session |
| Critique | `critique.md` | User wants structured feedback on someone's work |
| Pitch | `pitch.md` | User needs to build a persuasive case |
| Anticipate | `anticipate.md` | User needs to prepare for questions on a topic |
