# Synthesize

## Typical flow
1. **Collect sources** — Identify all inputs (files, references, prior research)
2. **Extract key points** — Pull the important claims, data, and insights from each source
3. **Identify themes** — Cluster related points into emerging themes or categories
4. **Cross-reference** — Note where sources agree, disagree, or fill each other's gaps
5. **Write synthesis** — Produce a thematic summary that weaves sources together

## Inputs
- Multiple sources: files, notes, references, or prior research outputs
- Optional: a guiding question or angle to synthesize around

## Outputs
- Thematic summary organized by patterns, not by source
- Source attribution for each major point
- Conflicts or tensions between sources noted explicitly

## Quality signals
- Organized by theme, not by source — the reader sees patterns, not a list of summaries
- Every claim traces back to at least one source
- Contradictions between sources are surfaced, not hidden
- The synthesis adds structure the individual sources lacked

## Common variations
- Literature review (academic or industry sources)
- Meeting synthesis (combine notes from multiple conversations)
- Project synthesis (pull together scattered project docs into one picture)
- Progressive synthesis (update an existing synthesis with new inputs)
